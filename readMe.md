# Инструкция по использованию шаблона MRAID

Добро пожаловать! Эта инструкция поможет вам создать HTML-креатив для рекламы, даже если вы никогда не программировали.

---

## 📋 Подготовка

Перед началом работы необходимо установить несколько программ:

### 1. Установка Cursor
- Скачайте Cursor по ссылке: https://cursor.com/download
- Установите программу на свой компьютер
- Зарегистрируйтесь (подписка не требуется, достаточно бесплатной регистрации)

### 2. Дополнительные инструменты

**Если у вас Mac:**
- Откройте программу Terminal.app (стандартная программа на Mac)
- Выполните команду: `xcode-select --install`
- Следуйте инструкциям на экране

**Если у вас Windows:**
- Скачайте и установите Git: https://git-scm.com/install/windows
- Следуйте инструкциям установщика

---

## 🚀 Запуск

### Вариант 1: Клонирование репозитория (если еще не клонировали)

Если вы впервые работаете с этим шаблоном:

1. Откройте Cursor
2. Нажмите кнопку **"Clone Repo"** (Клонировать репозиторий)
3. Вставьте ссылку: `https://github.com/FlingApp/template_playable_4.git`
4. Нажмите кнопку **"Clone"**
5. Выберите папку на компьютере, где будет храниться проект (запомните это место!)
6. Cursor предложит открыть репозиторий.

### Вариант 2: Открытие уже клонированного репозитория

Если репозиторий уже был клонирован ранее:

1. Откройте Cursor
2. В меню выберите **File → Open Folder** (Файл → Открыть папку)
3. Найдите и выберите папку с проектом (ту, куда вы клонировали репозиторий ранее)
4. Нажмите **"Открыть"** (или **"Open"**)

## 🔄 Перед началом работы

Каждый раз перед работой обновляйте шаблон до последней версии:

1. Откройте терминал внутри Cursor (обычно находится внизу окна)
2. Выполните команду: `git pull origin master`
3. Дождитесь завершения обновления

Эта команда загрузит самую актуальную версию шаблона с сервера.

## ✨ Основные возможности шаблона

### 1. Что можно редактировать

**✅ Редактировать можно только файл `base.html`** — в нём вы меняете стили (цвета, шрифты, размеры) и разметку креатива.

**❌ Файлы в папке `template/` редактировать нельзя!** Они используются сборщиком как есть.

### 2. Замена контента

Вы можете заменить следующие файлы в папке `resources/`, и они автоматически появятся в финальном креативе (после новой генерации с помощью команды `sh build.sh`):

- **resources/logo.png** — заменяет логотип
- **resources/banner.png** — заменяет баннер
- **resources/bg_image.png** — заменяет фоновое изображение всей страницы
- **resources/body.txt** — заменяет основной текст (основной контент)
- **resources/playstore_url.txt** — ссылка на приложение в Google Play
- **resources/appstore_url.txt** — ссылка на приложение в App Store

**Важно:** Сохраняйте те же имена файлов и расширения (.png для изображений, .txt для текста). Все файлы должны находиться в папке `resources/`.

Ссылки из `playstore_url.txt` и `appstore_url.txt` при сборке подставляются в финальный HTML (в том числе в логику открытия магазина — см. ниже). Меняете URL — пересобирайте креатив командой `sh build.sh`.

### 3. Поведение playable: интеракции и открытие магазина

Чтобы было проще понять, что увидит пользователь в рекламе:

**Кнопка «Continue Reading» (основная CTA)**  
При нажатии срабатывает событие `playable-cta-click`. Шаблоны в папке `template/` (Unity, AppLovin и т.д.) подключают к нему открытие приложения в магазине через MRAID (`mraid.open` и аналоги). Это основной сценарий перехода в стор.

**Авто-переход после нескольких интеракций**  
Дополнительно в `base.html` заложена логика: после **трёх** учтённых интеракций креатив **сам** пытается открыть ссылку на магазин (Google Play на Android, App Store на iOS — по данным браузера). Так пользователь может попасть в стор и без нажатия на кнопку, если он уже «поигрался» со скроллом или тапами.

Что считается интеракцией (каждый тип учитывается, сумма идёт к трём):

- **Скролл** — колёсико мыши или прокрутка области с текстом (панель с историей).
- **Свайп** — движение пальцем по экрану (`touchmove`).
- **Тап** — клик по экрану **вне** основной кнопки CTA.

Подряд идущие события одного жеста (например, много событий скролла за доли секунды) **склеиваются** в одну интеракцию короткой задержкой, чтобы три жеста не превратились в десятки «плюсов».

**Если пользователь нажал основную CTA**  
Как только сработало событие `playable-cta-click`, авто-открытие стора по счётчику **отключается**: магазин открывает обработчик кнопки. Так не получается двойного открытия.

**Где править порог и логику**  
Количество интеракций, задержка объединения жестов и сами URL в шаблоне задаются в `base.html` (скрипт с плейсхолдерами `{{GOOGLE_PLAY_URL}}` и `{{APPSTORE_URL}}`). Конкретные ссылки подставляются при `build.sh` из `resources/playstore_url.txt` и `resources/appstore_url.txt` — их не нужно дублировать вручную в коде.

### 4. Изменение стиля

Вы можете использовать встроенный ИИ Cursor для изменения стиля креатива **в файле `base.html`**:
- Изменить цвета текста
- Изменить шрифты
- Настроить размеры элементов
- И другие визуальные параметры

Просто опишите ИИ, что вы хотите изменить, и он поможет внести изменения.

### 5. Создание финального креатива

После замены файлов и внесения изменений нужно собрать финальный файл:

**На Mac:**
1. Откройте терминал внутри Cursor
2. Выполните команду: `sh build.sh`
3. Дождитесь завершения (если resources/body.txt длинный, то может занимать длительное время)

**На Windows:**
1. Откройте папку со скриптом `build.sh`
2. Запустите файл с помощью программы **Git Bash**
3. Если не сработало с первого раза — попробуйте запустить еще раз

**Результат:** В папке `export/` появится новая подпапка с датой и временем сборки, включая секунды (например, `export/2026-01-30_22-54-30`). Внутри неё — подпапки для каждого шаблона (unity, applovin, mintegral, google) с файлами **.html** и **.zip**. Старые экспорты не удаляются; при необходимости их можно удалять вручную.

## 📦 Результат

### Где найти готовый креатив?

Готовые креативы сохраняются в папке **`export/`**. Каждый запуск `build.sh` создаёт новую подпапку с датой и временем, включая секунды (например, `export/2026-01-30_22-54-30`), чтобы можно было делать несколько сборок в одну минуту. Внутри неё:

- `export/2026-01-30_22-54-30/unity/` — unity.html, unity.zip
- `export/2026-01-30_22-54-30/applovin/` — applovin.html, applovin.zip
- `export/2026-01-30_22-54-30/mintegral/` — mintegral.html, mintegral.zip
- `export/2026-01-30_22-54-30/google/` — portrait.html, portrait.zip, landscape.html, landscape.zip

В каждой подпапке — файл **.html** и архив **.zip** для загрузки в рекламные платформы. Старые экспорты остаются в `export/` — их можно удалять вручную при необходимости.

### Просмотр креатива

**Локально:** Откройте нужный .html (например, `export/2026-01-30_22-54-30/applovin/applovin.html`) в браузере (двойной клик по файлу)

## 🧹 После работы

Если вы хотите вернуть шаблон к исходному состоянию (удалить все ваши изменения):

1. Откройте терминал внутри Cursor
2. Выполните команду: `./reset.sh`

**⚠️ ВНИМАНИЕ!** Команда `reset.sh` стирает все ваши изменения и возвращает шаблон к базовому состоянию. Используйте её только если уверены, что хотите удалить все внесенные изменения.

## 💡 Полезные советы

- Сохраняйте копии важных файлов перед использованием `reset.sh`
- Регулярно обновляйте шаблон командой `git fetch; git pull origin master`
- Если что-то пошло не так, попробуйте запустить `build.sh` еще раз
- Для вопросов и помощи используйте встроенный ИИ Cursor
